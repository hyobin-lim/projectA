Git 터미널 명령어 정리 + 설명

vs에서 터미널 부르는 단축키
Ctrt `(백틱)

저장소 관련:
- git init → 현재 디렉토리를 Git 저장소로 초기화 (새 프로젝트 시작할 때)
- git clone <URL> → 원격 저장소를 로컬로 복제 (GitHub 등에서 내려받을 때)

변경 사항 관리:
- git status → 현재 작업 디렉토리 상태 확인 (수정/스테이징 여부)
- git add <파일명> → 특정 파일을 스테이징 영역에 추가
- git add . → 모든 변경 파일을 스테이징 영역에 추가
- git commit -m "메시지" → 스테이징된 변경 사항을 커밋 (메시지에 변경 내용 기록)

브랜치 관리:
- git branch → 브랜치 목록 확인
- git branch <브랜치명> → 새 브랜치 생성
- git checkout <브랜치명> → 해당 브랜치로 이동
- git checkout -b <브랜치명> → 브랜치 생성과 동시에 이동
- git merge <브랜치명> → 다른 브랜치를 현재 브랜치에 병합

원격 저장소:
- git remote -v → 연결된 원격 저장소 확인
- git push origin <브랜치명> → 로컬 브랜치를 원격 저장소에 업로드
- git pull origin <브랜치명> → 원격 저장소의 최신 커밋을 가져와 병합

로그 & 되돌리기:
- git log → 커밋 기록 확인 (작성자, 날짜, 메시지)
- git diff → 변경된 내용 비교 (수정된 코드 확인)
- git reset --hard <커밋ID> → 특정 커밋으로 되돌리고 이후 변경 사항 삭제 (주의: 데이터 손실 가능)
- git revert <커밋ID> → 특정 커밋을 취소하는 새로운 커밋 생성 (안전하게 되돌릴 때 사용)
